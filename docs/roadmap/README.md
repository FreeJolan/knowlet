# 路线图

Knowlet 按 Wedge 策略分阶段演进。能力层面三方向同源、相互增强;叙事层面阶段性聚焦。详见 [ADR-0001](../decisions/0001-wedge-learning-first.md)。

## 阶段总览

```
阶段一 (0→1, MVP~V1):  学习型主打
阶段二 (V1→V2):        横向补齐知识库能力
阶段三 (V2→V3):        引入情报推送
阶段四:                全能平台
```

## 阶段一 — MVP / V1:学习型主打

**Slogan:** "用 AI 把任何内容变成可复习的卡片"

**核心特性**

- OCR → 卡片(扫书/截图直接生成)
- SRS 复习引擎(基于成熟算法,如 SM-2 / FSRS)
- AI 出题(从 Note / Card 生成多形式题目)
- 错题回流(独立实体,加权 SRS)
- 纯文本 + Frontmatter + Git 底座

**显式不做(阶段一)**

- 双链与图谱(留给阶段二)
- 订阅源 / 定时拉取(留给阶段三)
- 多端同步的复杂方案(本地 + Git 即可)

**衡量是否可进入阶段二**

- 至少一类典型用户场景已被打通(例:考研专业课、语言词汇学习)
- 错题反哺真的产生了 Note 改进的效果,而不只是日志
- 用户日复习频次稳定,SRS 流跑得顺

## 阶段二 — V1 → V2:横向补齐知识库能力

用户复习一段时间后,自然会需要"我的知识总览"。此时:

- 推出双链 / 反向链接
- 推出标签系统 / 知识图谱可视化
- AI 主动提示连接(见 [`../design/organization.md`](../design/organization.md))
- Slogan 升级:"不仅是卡片工具,更是你的第二大脑"

阶段二是**被用户需求推着走**,而不是先做出来再找需求。

## 阶段三 — V2 → V3:引入情报推送

已有稳定的卡片复习与知识库基础后:

- 订阅源 / RSS / API 拉取
- AI 自动摘要 + 候选 Card 生成
- TTL + 重新校验机制(见 [`../design/architecture.md`](../design/architecture.md))
- 情报与学习强耦合:新情报自动变候选卡片,用户筛选后入库

## 阶段四 — 全能平台

三条能力相互强化:

```
情报拉取 → AI 生成卡片 → 用户复习 → 错题反哺知识库 → 知识库驱动更精准订阅
```

阶段四不再是"叠加新能力",而是把已有能力的反馈环路打满。

## 特性优先级原则

每条新特性进入路线图前,先过四个问题:

1. **服务于当前阶段的破局点吗?** 否 → 进入 backlog
2. **会损害三条核心原则吗?**(AI 可选 / 数据主权 / 插件化) 是 → 拒绝
3. **能用现有领域实体表达吗?** 否 → 思考是否需要新增实体,而不是塞进现有实体里
4. **拒绝它的代价是什么?** 如果代价是"失去某类用户但他们不在当前阶段画像里",可以接受

这套原则是为了避免"什么都想做"的早期陷阱,具体每条候选特性的判断结果会在 ADR 或 design 文档里记录。
