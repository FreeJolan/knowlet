## 二、报告评分指南

每个 case 都有 4 个评估维度,给 1-5 分(留空 = 没体验到):

- **功能(F)** = 跟说明书一致吗?有 bug 吗?
- **直觉(I)** = 不看说明也能知道怎么用吗?发现成本高吗?
- **视觉(V)** = 在纸感浅色的整体语境下舒服吗?突兀 / 拥挤 / 信息层级有问题?
- **恢复(R)** = 出错 / 想取消时,知道怎么撤回吗?

外加自由文字:**抱怨 + 惊喜 + 改进建议**。任何反应都有用,不必客气。

### Case 8 — Sediment 模态 ambient(ADR-0013 Layer A)

**功能简述**:在 chat 里聊了几轮 → 点 sediment(把对话存成笔记)→ 模态打开 → 标题字段上方有一个折叠的 `<details>`:**"可能相关的笔记 (3)"**。展开看到 3 条相似笔记的标题 + 预览,点击在新 tab 打开。**默认动作仍然是"保存为新笔记"**(无 merge 按钮)。

**用户故事**:
> 你跟 AI 聊了 RAG retrieval 的一些细节,准备 sediment 成笔记。但你不确定 vault 里是不是已经有一条类似的 — 万一是,你想合并而不是新建一条重复。

**步骤**:
1. 在 chat 跟 AI 聊几轮(任何主题)
2. 点 sediment 按钮(右栏 AI dock 或 chat focus)
3. 模态打开 + ambient 面板是否出现(默认折叠)?
4. 展开,3 条相关笔记是否合理?(看相关性,不必精准)
5. 点其中一条,模态是否关闭 + 那条笔记打开?(**不应自动 merge**)
6. 仔细看模态下方的 disclaimer:"纯供参考。默认动作仍然是「保存为新笔记」。" 这条文案对你足够清楚 / 多余 / 不够吗?

**评估**:F __ / I __ / V __ / R __

**自由感受**(默认折叠是否埋太深?展开后面板视觉重量合适吗?):

### Case 9 — Mining 草稿 critical take + hover quote(M7.3)

**功能简述**:配置一个带 `--critical-take` 的 mining task,跑一次后,新草稿的 body 里会多一段 `## Critical take`(模型有内容的锐评,不是客套话)+ 关键句被 `<q data-original="原文">译文</q>` 包裹,hover 显示原文。

**用户故事**:
> 你订阅了 RSS,knowlet 自动从中提取草稿。你之前觉得草稿过于"中性"(只总结不评价),想看 AI 对每条新闻 / 论文的客观锐评 + 原文 hover 核对。

**步骤**:
1. terminal: 配置一个 mining task(用 `--critical-take` 标志):
   ```
   uv run knowlet mining add --name "test" --rss "https://feeds.arstechnica.com/arstechnica/index" \
     --prompt "Summarize tech news" --critical-take
   ```
2. 跑一次:`uv run knowlet mining run-all`
3. web 切到 drafts focus(Cmd+Shift+D)
4. 看新草稿 body 是否有 `## Critical take` 章节
5. 锐评是不是 **content-grounded**(不是 "this represents an important step")?
6. body 里是否有带虚下划线的字(hover 应该出 tooltip 显原文)?
7. 中文 UI 下,英文文章 → 草稿是否中文 / 锐评是否中文 / `<q>` data-original 是否保留英文原文?

### Case 10 — Quiz scope picker + loop(M7.4 重头戏)

**功能简述**:Cmd+Shift+Q 进 quiz focus mode → scope 选 "手选笔记" 或 "按 tag" → 默认 5 题 → 一题一页提交答案 → AI 给 1-5 评分 + reason + missing 点。

**用户故事**:
> 你过去一周读了 5 条 RAG 相关笔记,但**不确定自己是不是真懂了**。你想要 active recall — 让 AI 出 5 题(不是 ctrl-F 题),你回答,AI 给参考答案 + 评分,你看自己哪里漏了。

**步骤**:
1. 打开任一笔记(让 quiz 自动 pre-seed scope)
2. 按 Cmd+Shift+Q → quiz focus mode
3. scope picker:
   - (a) 默认勾上的笔记是当前打开的那条吗?
   - (b) 多选 / 取消勾选体验如何?
   - (c) 切到 "按 tag" 模式 — tag 列表是否按频次排序?(应该看到你 vault 中最常用的 tag 在最上面)
   - (d) 高级设置展开,改 N=3 试试
4. 点开始 → 看 "出题中…" 是否流畅,~10s 出题
5. 第 1 题:
   - (a) 题面是否真的需要思考(不是 ctrl-F 能找的)?
   - (b) 输入答案,Cmd+Enter 提交
   - (c) 评分是否合理?(如果你随便答,应该 1-2;认真答,应该 3-4)
   - (d) reference answer 是否参考价值 OK?
6. "下一题" 直到第 5 题 → 看 summary 页

**自由感受**(scope picker 滚动 100+ 笔记顺手吗?题面密度合适吗?评分是不是太严格 / 太宽松?):

### Case 11 — Quiz summary + 不同意 + Cards 回流(M7.4)

**用户故事**:
> 上一个 case 走完了 quiz loop,你来到 summary。你发现:
> - 第 2 题 AI 给你 4 分但你觉得自己其实没真的懂,想 disagree
> - 第 4 题你完全没答出来(1 分),想做成 Card 之后练

**步骤**:
1. summary 页:看总分 / 答对几题 / disagree 计数
2. 找一题点 ✗ 不同意 → textarea 是否展开?写两句理由,确认
3. 之后看那题是否标记成 "你已标记不同意"
4. 错题(score < 3 的)是否默认勾选?
5. 改一两条勾选(比如把答对的题也勾上,或把错题去勾)
6. 点 "批量做成 Card" → toast 是否提示成功?
7. 关闭 quiz,左栏切 Cards focus(Cmd+Shift+R),看新 Card 是否出现 + tags 包含 `quiz`

### Case 13 — Web search 自动触发(M7.5)

**功能简述**:LLM 注册了 `web_search(query)` + `fetch_url(url)` 两个 tool。用户问实时信息时,LLM **自主**调用 web_search → 看 snippets → 选 1-2 条 fetch_url → 基于内容回答。chat history 显示 `· web_search(query="...")` trace 行。

**用户故事**:
> 你问 "transformers 库最新版本是多少?" — 这个 LLM 训练截止之后的事它不知道,但它**有 web search 工具**,期望它自己用。

**步骤**:
1. 不需要任何配置(默认走 DDG IA fallback)
2. 在 chat 问一个**确实需要实时信息**的问题 — 例如:
   - "请告诉我今天 Hacker News 头条是什么"
   - "查一下 langchain 库 npm / pip 最新版本号"
   - "最近一周 AI 领域有什么新闻"
3. 观察 chat history:
   - LLM 是否真的调用了 web_search?(看 trace 行)
   - 是否后续调用了 fetch_url?(理想 1-2 次)
   - 回答是否基于搜索内容(不是泛泛而谈)?
4. 试问一个**不需要 web search 的问题**(常识题,如 "解释一下 RAG"),LLM 应该**不调** web_search
5. (可选)切到 chat focus(Cmd+Shift+C),trace 行是否同样显示?
6. (可选)切到 CLI:`uv run knowlet chat`,问同样的问题,是否同样工作(ADR-0008 parity)?

**评估**:F __ / I __ / V __ / R __
