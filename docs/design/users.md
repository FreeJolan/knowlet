> 活文档。复合画像可能随产品成熟演化。

## 共性画像

Knowlet 服务的核心人群是**知识工作者**:他们既要在某个领域做长期的深度学习(读论文 / 写代码 / 做研究 / 系统学一门技术),又会偶尔进入需要结构化记忆的场景(外语 / 备考 / 专业概念辨析)。

这两类活动在他们日常并不分裂,而是并行存在。**目前没有工具同时服务这两条线** —— Anki 只管复习不管研究,Obsidian 只管笔记不主动召回,ChatGPT memory 不持久也不属于用户。

## 复合画像的特征

| 维度 | 描述 |
|---|---|
| **职业 / 身份** | 程序员 / 研究员 / 技术人 / 持续学习的知识工作者 |
| **AI 使用** | 已习惯使用 Claude / Cursor / ChatGPT 等 AI 工具,**重度但偏好工具自带 LLM 而非新工具自带 AI** |
| **数据主权敏感度** | 高 —— 在意"我的笔记 / 对话 / 学习数据归我所有" |
| **付费意愿** | 愿意为 LLM API 付费,愿意为开源工具自托管付出小成本 |
| **学习模式** | 桌面为主(深度阅读 / 写作),移动端用于碎片复习与提问 |
| **挫败来源** | 现有 PKM 工具的整理负担、AI 工具的会话健忘、跨工具记忆割裂 |

## 阶段一服务的三个真实场景

详见 [ADR-0003](../decisions/0003-wedge-pivot-ai-memory-layer.md) 场景 A / B / C:

- **A. 研究 / 论文阅读** —— 桌面为主,周期性高强度
- **B. 信息流订阅与整理** —— 后台定时,用户审查在零散时间
- **C. 结构化重复记忆 + AI 增强** —— 复习高频(可移动端),写作类任务低频但重要

三个场景共享**同一份用户上下文**(目标 / 偏好 / 错误模式 / 词汇掌握),AI 在跨场景之间累积理解,这是 knowlet 区别于现有工具的核心差异化。

## 显式不服务的人群(阶段一)

- **团队 / 协作场景的用户**:knowlet 终生不做多用户([ADR-0003](../decisions/0003-wedge-pivot-ai-memory-layer.md) 的 B1)
- **不愿配置 LLM 的初阶用户**:阶段一首次配置非零摩擦([ADR-0005](../decisions/0005-llm-integration-strategy.md))
- **重度移动端用户**:阶段一以桌面 + PWA 为主,原生移动体验推到阶段二
- **应试 / 语言学习的纯 SRS 用户**:Anki / 多邻国等已经称手,knowlet 阶段一不试图取代它们的核心场景

## 用户角色的"决策视角"用法

阶段一所有产品取舍以这个复合画像作为**决策视角**(见 [ADR-0003](../decisions/0003-wedge-pivot-ai-memory-layer.md) 中的"破局点"讨论)。例:

- "默认 SRS 参数偏激进还是温和?" → 知识工作者的复习强度通常温和(不像考研生),所以 FSRS 默认参数偏温和
- "首次启动重建索引该不该让用户等?" → 对程序员可接受 30 秒~1 分钟,有进度提示即可
- "卡片正面默认显示 / 隐藏提示?" → 写作批改场景下需要 AI 看到原句,所以 Card 内容默认对 LLM 可见

具体决策逻辑写在对应的 ADR / design 文档中,本文档只描述"画像"。
