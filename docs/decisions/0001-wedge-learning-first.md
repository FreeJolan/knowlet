# 0001 — 选择"学习型"作为 Wedge 破局点

- Status: Accepted
- Date: 2026-04-30

## Context

Knowlet 的能力图谱可以同时覆盖三个方向:

1. **学习型** —— 类 Anki + ChatGPT + Readwise 的融合,偏个人成长
2. **情报型** —— 定时拉取 + 自动摘要 + 知识沉淀,偏研究/DevRel
3. **知识库型** —— 类 Obsidian + AI Agent,偏通用 PKM

如果三个方向同时作为对外叙事并行推进,会面临四个现实问题:

| # | 理由 | 说明 |
|---|---|---|
| 1 | 用户心智只能记住一个标签 | Obsidian = 双链,Anki = SRS,Readwise = 高亮。没有成功 PKM 产品靠"全能"出圈 |
| 2 | 早期研发资源有限 | MVP 阶段只够"做深一件 + 做浅两件",全做深则每件只有 60 分 |
| 3 | 功能优先级会互相冲突 | 学习型要 SRS、情报型要订阅、知识库型要图谱,架构决策容易内耗 |
| 4 | 获客社群不同 | 学习型→考证社群;情报型→研究员社群;知识库型→PKM 社群。冷启动无法同时打三个 |

## Decision

采用 **Wedge Strategy(楔子战略)**:能力层面三方向同源、相互增强,但**叙事层面以"学习型"为唯一破局点先猛攻**,站稳后按下列顺序横向扩张:

```
阶段一(0→1, MVP~V1):学习型主打
  Slogan: "用 AI 把任何内容变成可复习的卡片"
  核心:  OCR 卡片 + SRS + AI 出题 + 错题回流
  副产品: 纯文本知识库(底层天然形成)

阶段二(V1→V2):横向补齐知识库能力
  推出双链、标签、图谱
  Slogan 升级: "不仅是卡片工具,更是你的第二大脑"

阶段三(V2→V3):引入情报推送
  订阅源 + 每日报告
  情报自动变候选卡片,与学习强耦合

阶段四:全能平台
  情报 → AI 生成卡片 → 复习 → 错题反哺 → 知识库驱动更精准订阅
```

之所以选学习型而非另两条作为破局点,有三个理由:

1. **OCR + 卡片 + 记忆科学**组合,目前市面上无成熟开源方案
2. **错题回流驱动 AI 加权出题**是最独特的设计,学习场景最能放大其价值
3. 其他两条能力**隐藏在底层**而非丢掉:
   - MD 底座 → 知识库能力天然就在,暂不作为对外主打
   - 定时拉取(Path 3) → 情报能力也在,先用来喂卡片,不单独宣传

## Consequences

**好处**

- 叙事聚焦,首批用户画像清晰(考研/考证/语言学习社群)
- MVP 范围可控,架构决策有明确锚点(围绕"卡片+SRS+错题"做深)
- 后续扩张是**被用户需求推着走**的自然演进,而非凭空铺面

**代价 / 约束**

- 前期需克制:即便能做的"知识库/情报"特性,如果不服务于学习场景就先不做对外主打
- 架构上仍需为后续阶段留出扩展点(MD 底座、能力插件化、领域实体一等公民等),否则阶段二再补时会撞墙
- 早期不要做"全能 PKM"的对外宣传,避免心智模糊

**类比**

Notion 早期也是全能(笔记+数据库+wiki+任务),但靠 "all-in-one workspace" 这个聚焦的口号打穿"讨厌切换工具的创业团队"细分人群,站稳后才横向扩张。Knowlet 同理:**能力是全的,但故事必须是窄的。**
