# 0003 — Wedge 破局点修订:AI 长期记忆层 + 减负型 PKM

> [English](./0003-wedge-pivot-ai-memory-layer.en.md) | **中文**

- Status: Accepted
- Date: 2026-04-30
- Supersedes: [0001](./0001-wedge-learning-first.md)

## Context

[ADR-0001](./0001-wedge-learning-first.md) 选择"学习型"作为 Wedge 破局点,具体描述为 OCR + 卡片 + SRS + AI 出题 + 错题回流,获客主打考证 / 语言学习社群。

在后续产品形态讨论中,ADR-0001 的"学习型"框架被识别出实际混合了两种相互拉扯的产品形态:

| 形态 | 描述 | 典型用户 |
|---|---|---|
| **A** | 应试 / 语言型 SRS 工具:强复习节奏、强卡片、强移动端 | 考证 / 语言学习社群 |
| **B** | 知识工作者的主动学习记忆体:强 RAG、强笔记沉淀,卡片为副 | 程序员 / 研究员 / 知识工作者 |

ADR-0001 的描述偏 A,但 [ADR-0002](./0002-core-principles.md) 的开源调性 / 数据主权 / 能力插件化原则更指向 B。两者并行会造成架构摇摆与心智模糊。

此外,在重新定位中浮出一个更深层的产品命题:

> 个人知识库的真正障碍不是"AI 不够强",而是**管理成本太高**。Obsidian / Notion / Roam 等工具的高放弃率主要来自:总结 / 分类 / 沉淀的心智负担、学习工具规则的迁移成本、以及"花了时间整理但用不上"的挫败感。
>
> LLM 真正的产品价值是**吃掉这些低 ROI、机械化的整理工作**,让用户的注意力集中在思考、提问、判断这些 AI 替不了的高 ROI 环节。AI 不只是"成本吸收器",同时也是"价值放大器"——它让旧笔记被自动连接、被主动召回、被跨场景累积。

基于以上两个判断,ADR-0001 的破局点叙事需要被替换。

## Decision

### 破局点定位

破局点切到形态 B,定位收紧为:

> **个人知识库 + AI 长期记忆层。** AI 替用户承担机械整理(总结、分类、沉淀、检索),用户保留意图、思考、判断;同时,任何 AI 工具(Claude / Cursor / 其他)在跟用户对话时,都可以从这个知识库主动检索用户私人的累积知识 —— 不仅在 knowlet 内可见,也在用户的所有 AI 工作流中可见。

### Slogan

- 中文:**"会自己整理的个人知识库"**
- 英文:**"A personal knowledge base that organizes itself."**

### 命名调性

仓库名 **knowlet** = `know + -let`(类比 booklet / droplet),意为 *a small unit of knowing*。

隐含调性:**谦逊、原子性、累积**。文档与产品文案不号称"第二大脑"、"AI 知识引擎"、"革命性"等大词,而是从"小知识的累积"切入。

### 阶段一服务的真实使用场景

#### 场景 A — 研究 / 论文阅读
用户主动找到论文 → 在 knowlet 嵌入式 chat 中讨论 → AI 写 Note 初稿 → 用户审查保存 → 沉淀。后续在任何 AI 对话中由 LLM 主动检索这条 Note,带历史结论一起回答。

#### 场景 B — 信息流订阅与整理
用户配置"知识挖掘任务"(频率 + 来源约束 + Prompt)→ 定时执行 → 抓取过程透明可追溯 → 生成多条原子 Note + 索引 Note → 用户审查后入库。

#### 场景 C — 结构化重复记忆 + AI 增强
覆盖外语词汇、专业概念辨析、写作批改等需要长期记忆与反复练习的内容。SRS 子模块负责调度;AI 在复习中按需提供造句 / 解释 / 近义辨析,在批改类任务中基于"用户上下文"调整反馈深度与风格。

三个场景共享**同一份用户上下文**(目标 / 偏好 / 错误模式 / 词汇掌握),AI 在跨场景之间累积理解。

### 阶段一核心特性

- **嵌入式 chat**:LLM 由用户自带(OpenAI 兼容协议),knowlet 不代理对话流量
- **LLM-driven retrieval**:LLM 在每次对话中按需从私人知识库拉相关内容
- **知识挖掘任务**:定时 + Prompt + 来源约束 + 抓取过程透明
- **AI 草稿 + 人工审查**作为默认沉淀模式
- **SRS 子模块(FSRS)**:作为知识库内的"主动复习视图",非独立产品
- **分层用户上下文**:Markdown 意图层 + JSON 派生分析层 + SQLite 派生状态层
- **桌面端为主 + 移动 PWA 兜底**碎片场景

### 阶段一明确不做

- 团队协作 / 多用户(终生不做)
- 内容推荐 / 信息发现 / 社交
- 任务 / 日历 / Todo 管理
- 传统 PKM 的双链 / 图谱专门 UI(改由 LLM agent + tools 间接达成)
- 跨用户的内容共享 / 知识包发布
- AI 自主决策(用户始终是决策者,AI 是执行者)
- AI Chat 产品的功能复刻(模型选择优化 / 长上下文 / 代码生成 / 图像生成)
- 内容审核 / 安全过滤
- 内置 Wikipedia / 公共知识库
- OT / CRDT 实时多端编辑
- knowlet chat 不抢 Claude / Cursor 的位置:只在"工作与沉淀相遇"时入场

### 阶段演进锚点

详见 [`../roadmap/`](../roadmap/)。简要:

- **阶段一(MVP / V1)**:以上核心特性
- **阶段二**:由用户自然需求驱动的扩展(图谱可视化 / plugin 生态 / 移动端原生体验)
- **阶段三 / 长远**:knowlet 的原子能力作为 MCP server,跨 AI 工具暴露,呼应"AI 长期记忆层"破局点

### 与其他 ADR 的关系

- **Supersede** [ADR-0001](./0001-wedge-learning-first.md) 的破局点叙事(Slogan / 核心特性 / 获客 / 阶段顺序),但保留其中"Wedge 战略"作为方法论 —— 即"能力同源、叙事窄"这一基本判断
- **从属于** [ADR-0002](./0002-core-principles.md) 三条核心原则
- **互补于** ADR-0004(AI 编排 + 原子执行哲学)/ ADR-0005(LLM 接入策略)/ ADR-0006(数据存储与同步)

## Consequences

### 好处

- **叙事聚焦**:目标用户清晰(知识工作者 / 程序员 / 研究员 / 持续学习者)
- **与 ADR-0002 调性完全对齐**:数据主权 / AI 可选 / 插件化的承诺自然成立,不再有原 ADR-0001 的张力
- **正面回应 PKM 高放弃率痛点**:把"管理成本"作为 AI 的主要服务对象,挫败感来源被消除
- **跨场景上下文累积形成天然差异化**:Anki / Claude / 多邻国 / Notion AI 都做不到这一点(它们彼此孤岛)
- **阶段二扩张是被用户需求推着走的延伸**,而非凭空铺面

### 代价 / 约束

- 放弃应试场景的强 SRS 流叙事(原 ADR-0001 的核心卖点之一)
- OCR + 卡片降级到 SRS 子模块,不再是阶段一主战场
- 用户需自带 LLM(OpenAI / Anthropic / Ollama 等),首次配置非零摩擦,但目标用户可接受
- 同步管道由用户自带(iCloud / Dropbox / Syncthing 等),knowlet 阶段一不内置同步逻辑
- LLM provider 看得到用户的对话 + RAG 命中片段;knowlet 不代理也不过滤,隐私由用户的 LLM 选择决定
- "重建机制"(从 Markdown / JSON 重建派生数据)在大型 vault 首次启动有可见延迟,可接受
